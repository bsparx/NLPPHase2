# Qwen3.5-0.8B Fine-Tuning Pipeline
## NLP Assignment 04 — Track 1, Option A (SFT → DPO)

This repository contains a complete, automated pipeline to fine-tune **Qwen/Qwen3.5-0.8B-Base** using **Modal.com** GPUs, with test-set generation via **DeepSeek API**.

---

## 📁 File Overview

| File | Purpose | Runs On |
| :--- | :--- | :--- |
| `generate_test_set.py` | Generates 10 prompts + gold answers via DeepSeek | Local |
| `modal_train.py` | Complete Modal app: 5 SFT trials → evaluate → 5 DPO trials | Modal (GPU) |
| `evaluate.py` | Generates report tables, qualitative examples, Markdown skeleton | Local |
| `test_set.json` | The 10 test prompts + gold answers (generated) | — |
| `sft_results.json` | SFT trial scores + predictions (generated by Modal) | — |
| `dpo_results.json` | DPO trial scores + predictions (generated by Modal) | — |
| `REPORT_SKELETON.md` | Final report template (generated by evaluate.py) | — |

---

## 🚀 Quick Start

### Step 0 — Prerequisites

Make sure you have:
- Python 3.10+ on your local machine
- `modal` CLI installed and authenticated (`modal setup` done)
- A Modal.com account with GPU quota (A10G)
- DeepSeek API key (already embedded in `generate_test_set.py`)

Install local dependencies:
```bash
pip install openai
```

### Step 1 — Generate the Test Set

```bash
python generate_test_set.py
```

This calls DeepSeek-v4-flash to:
1. Generate 10 diverse instruction prompts
2. Generate gold (reference) answers for each
3. Save everything to `test_set.json`

**Expected time:** ~1–2 minutes  
**Cost:** Negligible (DeepSeek flash tier is cheap)

### Step 2 — Run the Full Pipeline on Modal

```bash
modal run modal_train.py
```

This single command will:
1. Upload `test_set.json` to Modal
2. Spin up an A10G GPU worker
3. Run **5 SFT trials** on `yahma/alpaca-cleaned` (5k samples)
4. Evaluate each SFT trial on the 10 test prompts (BLEU + BERTScore)
5. Pick the best SFT model automatically
6. Run **5 DPO trials** on `trl-lib/ultrafeedback_binarized` (2k pairs) starting from the best SFT adapter
7. Evaluate each DPO trial
8. Save all adapters + results to the Modal volume `qwen-adapters`
9. Print a complete results table locally

**Expected time:** ~4 hours of GPU compute  
**Your active time:** ~30 seconds (just run the command)

> 💡 **Pro tip:** The script runs overnight. You can close your laptop. Modal keeps the job running.

### Step 3 — Download Adapters (Optional)

If you want to play with the adapters locally or need them for your report appendix:

```bash
mkdir -p local_adapters
modal volume get qwen-adapters / local_adapters/
```

### Step 4 — Generate the Report Skeleton

```bash
python evaluate.py
```

This reads the local `sft_results.json` and `dpo_results.json` (downloaded automatically by Modal) and creates:
- `REPORT_SKELETON.md` — a complete Markdown report with all tables, qualitative examples, and analysis sections

### Step 5 — Convert to Word / PDF

```bash
# Option A: Pandoc (recommended)
pip install pandoc
pandoc REPORT_SKELETON.md -o Report.docx

# Option B: Copy-paste
Open REPORT_SKELETON.md in any Markdown viewer, then copy-paste into Microsoft Word.
```

## 📊 What Gets Evaluated

For every trial, the following metrics are computed on the **10 test prompts**:

| Metric | Library | What it measures |
| :--- | :--- | :--- |
| **BLEU** | `sacrebleu` | N-gram overlap with gold answer |
| **BERTScore** | `bert-score` (DistilBERT) | Semantic similarity via embeddings |
| **Val Loss** | TRL Trainer | Training/validation loss (tie-breaker) |

**Selection rule:** Maximize `BLEU + BERTScore`. If tied, pick the one with **lower validation loss**.

## 🛠️ Troubleshooting

### "test_set.json not found"
Run Step 1 first:
```bash
python generate_test_set.py
```

### Modal job times out
The script is set to 3 hours per stage. If it times out, you can run SFT and DPO separately by modifying `modal_train.py` to comment out the DPO call, running SFT first, then uncommenting and running DPO.

### Out of Memory (OOM) on A10G
Qwen3.5-0.8B is tiny (0.8B params). OOM is unlikely. If it happens:
- Reduce `MAX_SEQ_LENGTH` to 256 in `modal_train.py`
- Reduce `batch_size` further
- Use `gradient_accumulation_steps` to maintain effective batch size

### BERTScore is slow
BERTScore downloads a DistilBERT model on first run. On Modal this is cached in the image. Locally, it will download once (~100MB).
